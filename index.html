Web Design

은행 웹사이트 디자인 2021년

프로젝트 개요:

사용자 친화적인 웹디자인을 기반으로 한 가상의 모바일 은행 웹사이트 프로젝트입니다. 이 프로젝트는 사용자 경험을 최우선으로 고려하여 설계되었습니다.

프로젝트 인원: 1명 (혼자 제작)

사용 도구: Figma

Project Overview:

This project is a virtual mobile banking website based on user-friendly web design. It was designed with a focus on user experience.

Team Size: 1 person (individually developed)

Tools Used: Figma

초기 사용자 리서치 단계에서는 현재 의료 시스템의 디지털화 과정에서 발생하는 핵심 문제점들을 파악하는 데 중점을 두었습니다. 특히, 고령 사용자들의 디지털 접근성 문제가 가장 큰 과제로 대두되었습니다.

프로토타입 테스트 결과, 65세 이상 사용자 그룹의 태스크 완료율이 기존 40%에서 85%로 크게 향상되었습니다. 음성 안내 기능과 고대비 모드를 추가하여 WCAG 2.1 AA 기준을 충족하는 접근성을 확보했습니다.

Design & Development

CodeEasy 2024

프로젝트 개요:

저시력자를 위한 코딩 알고리즘 학습 웹사이트입니다. 이 웹사이트는 저시력자를 고려하여 컬러감을 강조하고, 접근성을 최우선으로 설계되었습니다.

프로젝트 인원: 프론트엔드 팀 3명, 백엔드 팀 3명

사용 도구: Figma, React

Project Overview:

This website is designed for visually impaired individuals to learn coding algorithms. It emphasizes color schemes that are accessible for low-vision users and prioritizes usability.

Team Size: 3 frontend developers, 3 backend developers

Tools Used: Figma, React

이 프로젝트에서는 다양한 연령대의 학습자들을 위한 맞춤형 교육 경험을 제공하기 위해 리서치를 진행했습니다. 특히, 온라인 학습의 접근성과 사용자 경험을 개선하는 데 중점을 두었습니다.

사용자 테스트 결과, 학습자들이 플랫폼을 통해 더 나은 학습 경험을 얻을 수 있었으며, 피드백을 통해 지속적으로 개선할 수 있는 기회를 발견했습니다.

Design & Development

GroovyMap 2024

프로젝트 개요

음악, 댄스 등 각 공연예술 분야의 사람들을 위한 커뮤니티 웹앱으로, 지도를 활용한 공연 연습, 거리 공연장 찾기 및 커뮤니티 기능이 합쳐진 사이트

프로젝트 인원: 프론트 3명, 백엔드 3명

프로젝트 기여도: 전체적인 UI/UX 설계, 메인 화면, 로그인 회원 가능 페이지 디자인 및 기능 구현, 공연 장소 및 연습 장소 구현, 지도 API 사용

사용 도구: Figma, Next.js

Project Overview

This is a community web app for people in various performing arts fields such as music and dance, combining features for rehearsal, finding street performance venues, and community functions.

Team Size: 3 frontend developers, 3 backend developers

Contributions: Overall UI/UX design, main screen, design and implementation of login member pages, implementation of performance and rehearsal venues, use of map API.

Tools Used: Figma, Next.js

여행 플랫폼 프로젝트에서는 사용자들이 여행 계획을 세우는 데 필요한 모든 정보를 쉽게 찾을 수 있도록 하는 데 중점을 두었습니다. 사용자 인터뷰를 통해 그들의 요구사항을 파악하고, 이를 바탕으로 디자인을 개선했습니다.

프로토타입 테스트를 통해 사용자들이 플랫폼을 쉽게 탐색할 수 있도록 하였으며, 피드백을 통해 지속적으로 개선할 수 있는 기회를 발견했습니다.
